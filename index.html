3. 名词解释

卡池
游戏中角色的抽取池，通常有时间限制和概率提升机制
上下半
游戏版本更新周期内划分的两个时间段，通常各有一个提升角色抽取概率的卡池
周边
与游戏相关的实体商品，如手办、服饰、立牌、徽章等

4. 产品概述

鸣潮情报站是为《鸣潮》游戏玩家提供信息资讯的平台，包含小程序端和后台管理系统两部分。
小程序为用户提供卡池、活动和周边资讯查询功能，后台管理系统为管理员提供内容管理功能。

5. 小程序功能需求

5.1 卡池页面

展示《鸣潮》游戏历史角色卡池信息

  • 以卡片形式展示每个卡池
  • 显示卡池名称、版本号、上下半信息
  • 显示卡池时间范围
  • 展示卡池角色的精美图片

5.2 活动页面

展示《鸣潮》游戏所有活动信息。

  • 按时间顺序排列活动
  • 显示活动名称、图片、时间范围
  • 点击查看活动详情
  • 清晰展示活动是否正在进行

5.3 周边页面

展示《鸣潮》游戏相关周边资讯。

  • 展示官方周边产品信息
  • 展示联动周边活动信息
  • 显示周边名称、图片、发布时间
  • 可跳转到购买链接

6. 后台管理系统功能需求

6.1 卡池历史管理

6.1.1 添加卡池

管理员可以添加新的卡池信息，包含以下字段：

字段名 类型 必填 说明
id 数字 默认创建 卡池唯一标识
name 文本 是 角色/武器名称
imageUrl URL 是 角色图片链接
version 文本 是 版本号(如1.0, 1.1等)
position 单选 是 上半/下半
startDate 日期 是 卡池开始日期
endDate 日期 是 卡池结束日期

6.1.2 卡池列表

  • 以最新版本排序展示所有卡池
  • 支持编辑和删除操作

6.2 活动管理

6.2.1 添加活动

管理员可以添加新的活动信息，包含以下字段：

字段名 类型 必填 说明
id 数字 默认创建 卡池唯一标识
name 文本 是 活动名称
imageUrl URL 是 图片链接
articleUrl URL 是 文章链接
startDate 日期 是 活动开始日期
endDate 日期 是 活动结束日期

6.2.2 活动列表

  • 以最新排序展示所有活动
  • 支持编辑和删除操作

6.3 周边管理

7. 技术需求

7.1 小程序技术栈

  • 微信小程序原生开发或Taro框架
  • WXML/WXSS/JavaScript
  • 使用微信云开发或自建后端API

7.2 后台管理系统技术栈

  • 前端: Vue.js/React + Element UI/Ant Design
  • 后端: Node.js/Java/Python等
  • 数据库: MySQL/MongoDB
  • 部署: 阿里云/腾讯云等云服务

8. 项目时间线

阶段 时间 交付物
